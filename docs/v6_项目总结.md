## 一、项目概述

### 1.1 起点

ETF轮动策略 v4/v5 存在一个确认的失效模式：在"权益牛市+黄金横盘/下跌"的市场环境中产生深度负收益（2017年：-17.9% vs 沪深300 +21.8%；2026年YTD初步数据：-8.3%）。三个根本原因：

1. **横截面打分**：当所有ETF的绝对趋势都弱时，仍强制选出"最不差的"ETF
2. **ADX后期偏向**：40%权重放在无符号趋势持续性上，偏好即将结束的黄金趋势而非新兴的权益趋势
3. **无处轮动**：黄金趋势缺失时，6只ETF的池子中没有替代资产可配置

## 二、架构演进

### 2.1 架构A：混合打分（保留作为备用）

**文件**: `scripts/_funcs.py`（582行）+ `scripts/asset_rotation_v6.py`

四分量混合打分模型：LFMM趋势+ADX归一化 + 横截面动量+加速度z-score。7只ETF的月度Top-2选择，逆波动率加权，宏观四象限制度检测（PMI + 国债收益率方向），动态黄金上限，回撤控制。

**OOS Sharpe**: 0.517 | **可接受标准**: 9/17 PASS

**核心问题**：逆波动率加权系统性地将50-80%的资本分配给债券/现金，无论权益得分多高。月度Top-2选择在7只ETF中产生700%+的年换手率。任何参数干预（黄金上限、稳定性偏好、得分门限调整）都会降低Sharpe——系统已处于局部最优。

### 2.2 架构B：PE-Band + 黄金趋势（主要策略）

**文件**: `scripts/v6_pe_band.py`（493行，自包含）

两个信号，三只ETF，月度再平衡：

1. **PE-Band**: `权益仓位 = 60% - PE分位数 × (60% - 10%)`（分位数越低=越便宜=仓位越高）
2. **黄金趋势**: `黄金仓位 = 30%（若金价>MA50且MA50斜率>0）否则 0%`

债券获得剩余仓位。无逆波动率加权、无制度检测、无得分增强、无横截面归一化。

**OOS Sharpe**: 1.249 | **可接受标准**: 11/12 applicable PASS

**相对于架构A的优势**:
- Sharpe +76%
- CAGR +2.9倍
- 最大回撤改善5个百分点
- 换手率降低34倍（168% vs 705%）
- 代码量减少4.6倍（493行 vs 581行+200行）

### 2.3 架构B为何胜出

三个结构性原因：

1. **PE分位数是真正的反周期信号** — 估值均值回归是中国权益市场的结构性特征。PE-Band直接表达了这个逻辑。
2. **无逆波动率拖累** — 仓位由估值决定，不由波动率决定。便宜时获得足额权益敞口。
3. **二元黄金过滤器消除轮动噪音** — 黄金要么趋势中（30%），要么不是（0%）。不与权益竞争得分，无月度重新排名噪音。

## 七、经验总结

### 7.1 简单胜过复杂——而且差距可以量化

架构B用125行代码在两个信号上击败了581行代码的4分量混合系统。逆波动率加权是架构A的结构性瓶颈——它系统性超配债券/现金，无论信号质量如何。PE-Band按估值分配仓位，不受波动率影响。更简单的架构有更少的组件间干扰点。

### 7.2 跨折收敛的参数是真正的信号

PE-Band模型的参数在所有三个独立折中都收敛到了相同的值。架构A的校准从未真正收敛——不同的Executor在相同的网格搜索中得出不同的结果。当参数在不同训练窗口中保持稳定时，模型捕捉到的是结构性的东西。当它们不稳定时，你在拟合噪声。这是一个比任何单一指标都更好的验证标准。

### 7.3 最有影响力的设计决策是无形的——直到它不是

整个环节中最大的性能改进来自将PE分位数窗口从全历史（20年）切换到5年滚动。这使折2的Sharpe从-0.39变为+0.67。没有人计划这个——它来自于消融测试发现2007年泡沫PE 50在扭曲2024年的分位数。全历史窗口是默认选项，因为它看起来显然正确——更多数据意味着更好。但默认假设是错的。

### 7.4 宏观叠加一致增加噪声

测试了三个宏观扩展：PMI过滤、制度自适应黄金MA、以及完整的四象限制度检测。全部降低了性能。PE信号已经隐含地编码了宏观条件——便宜的市场与经济疲软相关，昂贵的市场与经济强劲相关。添加显式宏观变量引入了测量误差，而没有增加信号。

### 7.5 Executor代理会覆盖你的修复

在本次会话中，Executor代理三次覆盖了我刚刚应用的修复。模式总是一样的：修复`_funcs.py`中的Bug，为下一阶段生成Executor，Executor读取文件，根据自己对计划的理解从头重写，重新引入旧Bug。v4的`backtest()`签名被覆盖了四次，`T0`/`T1`常量被删除和重建了三次，CASH按价值存储的修复被还原了两次。修复要在下一个Executor的提示中包含为显式约束，或者在修复后立即提交。

### 7.6 源码删除是多代理工作流的真实风险

一度，所有四个关键的Python文件和大部分计划文档被同时删除。Executor在`scripts/`中创建了新文件然后自我清理——包括它们没有创建的文件。恢复是可能的，因为Git有提交，`.pyc`字节码缓存可以提取函数名和常量，计划规范和结果文件包含了足够的信息来重建。没有Git，这将是完全损失。

### 7.7 计划文档是最持久的产出

当源文件被删除时，`etf_rotation_v6_revised.md`中的计划是唯一最有价值的恢复资产。它包含了函数签名、参数表、打分公式、制度逻辑和接受标准——重建所需的一切。可以从规范重新生成代码；无法从代码重新生成规范。

## 八、限制与未来工作

### 已识别的限制

1. **折2（2022-2023）在所有配置中都为负收益** — PE信号在2022年初处于50分位数附近（"公允价值"），模型持有约30%权益。沪深300在2022年下跌21%。PE-Band在价格下跌时正确减少了权益，但无法完全避免回撤。这是估值驱动型配置的固有特性。

2. **债券久期风险** — 当前信号86%配置于国债ETF（~5年久期）。100bp的利率上行意味着约5%的净值下跌。回测期间中国利率持续下行——模型从未见过持续加息环境。

3. **单只权益ETF** — 模型仅配置沪深300。中证500和创业板被排除。在小盘或成长股大幅跑赢的年份，模型无法捕捉行业alpha。

4. **PE分位数在极端区间丢失信息** — 在92分位数（当前水平），权益仓位为14%。在99分位数，为10.5%。仓位在"昂贵"区间几乎无变化。模型在权益便宜时最为精确——这在最重要的时刻是正确的权衡。

5. **无宏观制度感知** — PE信号仅基于估值。不区分"因增长放缓而便宜"和"因恐慌抛售而便宜"。添加PMI过滤器未改善性能，但制度感知在小样本中（11年月度数据）本质上是困难的。
